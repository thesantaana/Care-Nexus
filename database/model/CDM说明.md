## 1. CDM范围

CareNexus 概念数据模型按业务领域划分为：

- 用户、角色、权限和账号；
- 老人档案、老人家属绑定和医生老人授权；
- 护理培训资源、学习记录、考核和 AI 草稿；
- 护理服务、地址、订单、执行记录、评价和投诉；
- 医生健康记录、健康预警、随访、干预和健康评估；
- 文件资源和操作日志。

## 2. 核心实体

| 领域 | 概念实体 |
|---|---|
| 用户权限 | 用户、角色、权限、字典、操作日志 |
| 关系授权 | 老人档案、老人家属绑定、医生老人授权 |
| 护理培训 | 培训类别、培训标签、培训资源、学习记录、学习访问记录、考核、题目、题目选项、考试记录、答案、AI草稿、AI草稿资料来源 |
| 护理服务 | 服务项目、服务地址、护理订单、订单状态日志、服务记录、评价、投诉 |
| 医生健康 | 健康记录、健康预警、随访记录、干预记录、健康评估 |
| 文件 | 文件资源 |

## 3. 关键关系

- 一个用户拥有一个主要业务角色，角色决定默认业务端入口。
- 一个老人账号可关联一个老人档案；老人档案也允许由管理人员先建档后绑定账号。
- 一个老人可被多个家属绑定，一个家属可绑定多个老人，但每对老人和家属只保留一条当前绑定记录。
- 一个医生可被授权访问多个老人，一个老人可授权给多个医生，但每对医生和老人只保留一条当前授权记录。
- 培训资源可关联多个标签，可被用户产生学习访问记录。
- 一个考核包含多个题目，一个题目可被多个考核复用。
- AI草稿必须基于一份或多份培训资源生成，草稿与资料来源通过关系实体记录，审核通过后才能形成正式题目。
- 护理订单关联老人、下单用户、服务项目、服务地址和护工；评价和投诉独立建模。
- 健康记录可触发健康预警；随访、干预和健康评估均关联授权老人和医生。
