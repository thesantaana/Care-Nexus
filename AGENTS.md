# CareNexus Project Context

CareNexus 颐联是一个面向智慧护理场景的四端互联管理系统。项目以护理培训系统为核心，扩展移动护理平台、医生服务系统和综合管理系统，通过统一账号、统一角色权限、统一老人档案、统一订单/服务数据实现跨端互联互通。

## MVP Boundary

MVP 阶段仅在护理培训系统中接入 AI 能力，聚焦培训资料问答、知识点总结、学习路径建议和练习题生成。医生端 AI 诊断、老人端 AI 问诊、第三方医院系统接口、真实支付和真实短信网关暂不纳入首期交付。

## Documentation Structure

项目过程文档统一放在 `docs/` 目录下：

- `docs/logs/`：个人工作日志，每位成员每天填写一条。
- `docs/meetings/`：会议记录，包括选题、需求、设计、测试、答辩准备等关键会议。
- `docs/plans/`：项目开发计划、项目进度计划、配置管理计划。
- `docs/requirements/`：软件需求规约、用例说明、业务流程、原型说明。
- `docs/design/`：系统架构设计、数据库设计、接口设计、概要设计、详细设计。
- `docs/test/`：测试计划、测试用例、缺陷跟踪表、测试报告。
- `docs/delivery/`：答辩 PPT、项目总结、部署说明、交付清单和最终汇总材料。
