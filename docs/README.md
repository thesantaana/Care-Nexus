# 项目文档目录说明

本目录用于统一管理 CareNexus 颐联项目的软件工程过程材料、设计材料、测试材料和交付材料。所有文档应按类别归档，避免散落在仓库根目录。

## 目录结构

```text
docs/
├── logs/              # 个人工作日志
├── meetings/          # 会议记录
├── plans/             # 项目计划、进度计划、配置管理计划
├── requirements/      # 需求规约、用例、业务流程、原型说明
├── design/            # 架构、数据库、接口、概要/详细设计
├── test/              # 测试计划、测试用例、缺陷跟踪、测试报告
└── delivery/          # 答辩PPT、项目总结、部署说明、交付清单
```

## 使用要求

- 每个目录中的 `README.md` 说明该目录应存放的内容。
- 每位成员每天填写个人工作日志，统一放入 `docs/logs/`。
- 会议记录、计划、需求、设计、测试和交付材料由对应负责人维护。
- 文档文件名建议包含日期或版本号，便于后续整合和验收追踪。

## 建议命名示例

```text
2026-07-08-隋咏轩-个人工作日志.md
项目开发计划-v1.0.docx
软件需求规约-v1.0.docx
测试用例-v1.0.xlsx
缺陷跟踪表-v1.0.xlsx
项目总结-v1.0.docx
```
