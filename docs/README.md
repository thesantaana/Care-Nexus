# 项目文档目录说明

本目录用于统一管理 CareNexus 颐联项目的软件工程过程材料、需求材料、设计材料、测试材料和交付材料。所有文档必须按唯一目录归档，避免同类材料重复存放。

## 最终统一目录结构

```text
docs/
├── plans/             # 项目开发计划、进度计划、配置管理计划
├── requirements/      # 软件需求规约、用例模型、关键用例、业务流程、MVP边界
├── design/            # 架构、概要/详细设计、图表、数据库设计、原型
├── api/               # 接口设计、统一返回结构、错误码、接口变更记录
├── test/              # 测试计划、测试用例、缺陷记录、测试日志、测试报告
├── delivery/          # 部署说明、用户手册、产品说明、答辩材料、交付清单
├── logs/              # 个人工作日志
├── meetings/          # 项目会议记录
├── management/        # 项目状态、任务、问题、模块归属、变更记录
├── decisions/         # 架构决策记录 ADR
└── workflows/         # 工作流程规范，例如需求澄清流程
```

## 唯一存放位置

- 个人工作日志只放入 `docs/logs/`。
- 会议记录只放入 `docs/meetings/`。
- 测试日志只放入 `docs/test/TEST_LOG.md`。
- 问题跟踪只放入 `docs/management/issues/ISSUE_TRACKER.md`。
- 单项任务文档只放入 `docs/management/tasks/`。
- 架构决策记录只放入 `docs/decisions/`。
- 阶段工件映射以 `docs/STAGE_INDEX.md` 为准。

## 使用要求

- 每个目录中的 `README.md` 说明该目录应存放的内容。
- 文档文件名建议包含日期、版本号或任务编号，便于后续整合和验收追踪。
- 不得再新建与上述目录重复的阶段目录或个人目录。
