# CareNexus Lite 软件需求规约

状态：与 `lite_develop` 当前成果一致

## 1. 目的与范围

系统服务于护理人员培训，由管理员维护课程与考核，护工完成在线学习和知识检验。首期只覆盖管理员与护工，不包含护理服务订单、老人家属、医生健康管理或医疗AI。

## 2. 角色

### 管理员

- 登录管理工作台。
- 管理分类、标签、课程资源、封面、文件、题库和考试。
- 发布或下架资源，发布考试并设置允许作答次数。
- 查看护工成绩，审核AI题目草稿。

### 护工

- 登录学习工作台并查看全部已发布课程。
- 阅读章节资料，形成学习访问记录和进度。
- 编写图文笔记，参与课程讨论、回复和点赞。
- 完成可重复作答的课程作业；完成满足学习条件的考试。
- 查看课程成绩、平均成绩、培训通过状态和错题。
- 使用基于课程资料的AI问答、总结、建议和练习。

## 3. 核心功能需求

| 编号 | 需求 |
| --- | --- |
| REQ-AUTH-001 | 用户通过账号密码登录，停用账号不得登录 |
| REQ-AUTH-002 | 系统返回当前用户、主要角色和权限码 |
| REQ-TRAIN-001 | 管理员维护分类和标签并控制启停 |
| REQ-TRAIN-002 | 管理员新增、编辑、发布和下架文章/视频/PPT资源 |
| REQ-TRAIN-003 | 课程支持封面、本地文件、文本或外部链接 |
| REQ-LEARN-001 | 护工只能学习已发布资源并记录访问、时长和最近学习时间 |
| REQ-LEARN-002 | 每门课程显示学习进度和考试成绩，全部课程通过才算培训通过 |
| REQ-NOTE-001 | 护工可按课程维护富文本及图片笔记 |
| REQ-DISCUSS-001 | 护工可发帖、回复、点赞和删除本人内容 |
| REQ-EXAM-001 | 题库和考试只支持单选题、判断题并自动评分 |
| REQ-EXAM-002 | 作业可无限重做；考试次数由管理员设置 |
| REQ-EXAM-003 | 考试前必须完成对应课程学习，60分及以上通过 |
| REQ-EXAM-004 | 保存考试记录、课程成绩、平均成绩和错题 |
| REQ-AI-001 | AI回答必须关联已发布培训资料并展示资料来源 |
| REQ-AI-002 | AI题目只生成草稿，经管理员审核后进入正式题库草稿 |
| REQ-FILE-001 | 图片和资料上传需要格式、大小、MIME和安全文件名校验 |

## 4. 状态与规则

- 资源：`DRAFT -> PUBLISHED -> OFFLINE`，下架不删除历史学习记录。
- 考试：`DRAFT -> PUBLISHED`；考试只含单选和判断题。
- AI草稿：`DRAFT -> APPROVED / REJECTED`，不可重复审核。
- 课程通过：对应考试最高成绩不低于60分；整体培训通过要求全部课程通过。
- 作业提交后立即自动批改，支持重做；考试受最大作答次数约束。

## 5. 非功能需求

- 权限：管理员和护工入口及操作隔离，后端每个受保护接口校验权限。
- 安全：BCrypt、JWT、统一401/403、敏感配置环境变量、文件上传安全校验。
- 可用性：三个前端均支持本地独立运行，门户提供统一入口。
- 可维护性：模块化单体、统一响应和异常、SQL脚本可重复初始化演示环境。
- AI安全：不提供诊断、处方、治疗或自动医疗决策；输出需提示核查重要信息。

## 6. 验收标准

管理员和护工演示账号可登录；课程管理、学习、作业、考试、成绩、错题、笔记、讨论和AI辅助均有可操作演示数据；后端和三个前端构建成功；未提交真实密码、Token或模型密钥。
