# 需求文档

适用基线：`lite_develop`，更新时间：2026-07-15。

本目录描述 CareNexus 轻量版要解决的问题、服务角色和业务闭环。

- `软件需求规约.md`：功能、规则、非功能需求和验收主线。
- `用例模型.md`：管理员、护工及外部系统用例。
- `业务流程.md`：登录、课程、考核、AI、笔记、讨论和作业流程。

当前角色只有管理员与护工。护理订单、老人家属、医生健康管理和医疗决策不属于轻量版。

需求变化应先更新 `LITE_SCOPE.md`，再同步用例、流程、架构、API、数据库和测试。原完整版需求基线保留在 Git 历史和过程记录中，不作为当前验收依据。
