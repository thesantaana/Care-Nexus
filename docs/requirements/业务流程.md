## 3. 培训资源发布流程

```mermaid
flowchart TD
    A["培训管理员登录护理培训系统"] --> B["维护培训类别和标签"]
    B --> C["新增或编辑文章、视频、PPT"]
    C --> D["保存为草稿"]
    D --> E{"内容是否可发布"}
    E -- "是" --> F["发布培训资源"]
    F --> G["资源状态：已发布"]
    G --> H["护工/护理人员可浏览学习"]
    E -- "否" --> D
    G --> I["后续需要下架"]
    I --> J["资源状态：已下架"]
    J --> K["历史学习记录保留"]
```

约束：

- 培训资源状态仅包含草稿、已发布、已下架。
- 下架资源不影响历史学习记录。

## 4. 培训学习和考核流程

```mermaid
flowchart TD
    A["护工/护理人员登录"] --> B["浏览已发布培训资源"]
    B --> C["学习文章、视频或PPT"]
    C --> D["系统记录整体学习时长和最近学习时间"]
    D --> E["用户参加培训考核"]
    E --> F["提交单选题、判断题和简答题"]
    F --> G["系统自动评分单选题和判断题"]
    G --> H["培训管理员人工评分简答题"]
    H --> I{"考核是否通过"}
    I -- "通过" --> J["整体培训状态：已通过"]
    I -- "未通过" --> K["整体培训状态：未通过"]
```

约束：

- MVP 阶段不按文章、视频、PPT 分别计算通过状态。
- 首期只记录用户整体培训状态。
- 简答题未评分时不得直接判定培训通过。

## 5. AI辅助培训与题目草稿生成审核流程

```mermaid
flowchart TD
    A["培训管理员上传或维护培训资料"] --> B["培训资料入库"]
    B --> C{"AI辅助类型"}
    C -- "培训资料问答" --> D["AI服务基于入库资料生成回答"]
    D --> E["直接展示辅助回答"]
    C -- "知识点总结" --> F["AI服务基于入库资料生成知识点总结"]
    F --> G["直接展示辅助总结"]
    C -- "学习建议" --> H["AI服务基于入库资料生成学习建议"]
    H --> I["直接展示辅助建议"]
    C -- "题目草稿生成" --> J["AI服务基于资料生成题目、选项、答案和解析草稿"]
    J --> K["系统保存AI生成草稿"]
    K --> L["培训管理员审核和修改草稿"]
    L --> M{"审核是否通过"}
    M -- "通过" --> N["进入正式题库或发布"]
    M -- "不通过" --> O["驳回草稿"]
```

约束：

- AI 仅用于护理培训辅助。
- 培训资料问答、知识点总结和学习建议作为辅助内容直接展示。
- AI 题目必须基于已上传或已入库培训资料生成草稿。
- 只有题目草稿必须经培训管理员审核后才能进入正式题库或考核。
- 不实现医生端 AI 诊断、AI 处方或自动医疗决策。

## 6. 老人和家属绑定流程

```mermaid
flowchart TD
    A["家属登录移动护理平台"] --> B["发起绑定老人"]
    B --> C["输入或选择老人识别信息"]
    C --> D["老人账号确认、绑定码或身份验证"]
    D --> E{"验证是否通过"}
    E -- "通过" --> F["建立老人和家属绑定关系"]
    F --> G["家属可查看已绑定老人列表"]
    G --> H["家属可代老人提交预约"]
    E -- "不通过" --> I["拒绝绑定"]
```

约束：

- MVP 阶段不需要管理员人工审核。
- 具体绑定交互方式在详细设计阶段确定。
- 未绑定的家属不得代老人提交预约。

## 7. 护理服务预约、分配、执行和完成流程

```mermaid
flowchart TD
    A["老人或家属登录移动护理平台"] --> B["浏览护理服务项目"]
    B --> C["维护或选择服务地址"]
    C --> D["选择老人、服务项目和预约时间"]
    D --> E{"信息是否完整"}
    E -- "否" --> F["拒绝提交预约"]
    E -- "是" --> G["提交护理服务预约"]
    G --> H["订单主状态：待分配"]
    H --> I["运营人员人工分配护工/护理人员"]
    I --> J["订单主状态：待确认"]
    J --> K["护工/护理人员查看分配订单"]
    K --> L["护工/护理人员确认服务"]
    L --> M["订单主状态：已确认"]
    M --> N["护工/护理人员开始服务"]
    N --> O["订单主状态：服务中"]
    O --> P["护工/护理人员完成服务并填写服务记录"]
    P --> Q["订单主状态：已完成"]
```

约束：

- MVP 不实现真实支付。
- MVP 不实现智能派单。
- 护工/护理人员只能处理分配给自己的订单。
- 完成服务必须填写简单服务记录。

## 8. 订单取消流程

```mermaid
flowchart TD
    A["老人或家属查看订单"] --> B{"订单主状态"}
    B -- "待分配" --> C["填写取消原因"]
    B -- "待确认" --> C
    B -- "已确认" --> C
    B -- "服务中" --> D["拒绝直接取消"]
    B -- "已完成" --> D
    C --> E["提交取消"]
    E --> F["订单主状态：已取消"]
```

约束：

- 老人或家属可以在服务开始前取消订单。
- 取消时必须记录取消原因。
- 服务中和已完成订单不得直接取消。

## 9. 评价和投诉处理流程

```mermaid
flowchart TD
    A["老人或家属查看订单"] --> B{"订单主状态是否已完成"}
    B -- "否" --> C["拒绝评价或投诉"]
    B -- "是" --> D{"用户选择操作"}
    D -- "评价" --> E{"评价状态是否未评价"}
    E -- "是" --> F["提交评价"]
    F --> G["评价状态：已评价"]
    E -- "否" --> H["拒绝重复评价"]
    D -- "投诉" --> I["提交投诉"]
    I --> J["投诉状态：处理中"]
    J --> K["运营人员处理投诉"]
    K --> L["记录处理结果"]
    L --> M["投诉状态：已处理"]
```

约束：

- 评价状态独立于订单主状态。
- 投诉状态独立于订单主状态。
- 评价和投诉可以独立发生。

## 10. 医生与老人授权流程

```mermaid
flowchart TD
    A["管理员或健康管理人员登录综合管理系统"] --> B["选择医生"]
    B --> C["选择老人"]
    C --> D["分配老人授权"]
    D --> E["记录授权操作日志"]
    E --> F["医生可查看授权老人健康数据"]
    F --> G{"是否取消授权"}
    G -- "是" --> H["取消医生与老人授权关系"]
    H --> I["记录取消授权操作日志"]
    I --> J["医生不得继续查看该老人健康数据"]
    G -- "否" --> F
```

约束：

- 医生只能查看授权老人。
- 授权维护参与者为管理员和健康管理人员。
- 医生本人只能使用已有授权，不能给自己新增授权。
- 授权变更必须记录操作日志。
- 授权取消后医生不得继续访问该老人健康数据。

## 11. 健康记录管理流程

```mermaid
flowchart TD
    A["医生选择授权老人"] --> B["查看历史健康记录"]
    B --> C["录入血压、血糖、心率、体温等指标"]
    C --> D["填写记录时间和备注"]
    D --> E["保存健康记录"]
    E --> F{"是否触发人工或基础阈值规则"}
    F -- "是" --> G["形成健康预警"]
    F -- "否" --> H["健康记录保存完成"]
```

约束：

- 医生只能管理授权老人健康记录。
- 健康记录属于隐私数据。
- 健康预警可由人工或基础阈值规则形成。

## 12. 健康预警处理流程

```mermaid
flowchart TD
    A["系统或人工产生基础健康预警"] --> B["预警状态：待处理"]
    B --> C["医生查看授权老人预警"]
    C --> D["医生开始处理预警"]
    D --> E["预警状态：处理中"]
    E --> F["医生填写处理说明"]
    F --> G["医生关闭预警"]
    G --> H["预警状态：已关闭"]
```

约束：

- 健康预警不等同于医疗诊断。
- 本期不设计 AI 诊断、AI 处方或自动医疗决策。
- 医生只能处理授权老人相关预警。

## 13. 随访、干预和健康评估流程

```mermaid
flowchart TD
    A["医生选择授权老人"] --> B["创建随访记录"]
    B --> C["填写随访方式、结果和备注"]
    C --> D["记录干预建议或干预措施"]
    D --> E["医生确认随访和干预记录"]
    E --> F["新增健康评估草稿"]
    F --> G["填写风险等级、评估结论和建议措施"]
    G --> H{"是否确认评估"}
    H -- "是" --> I["健康评估状态：已确认"]
    H -- "否" --> J["健康评估状态：草稿"]
```

约束：

- 随访和干预记录由医生确认后形成正式记录。
- 健康评估由医生确认后生效。
- MVP 不实现 AI 自动评估。

## 14. 综合管理用户和权限管理流程

```mermaid
flowchart TD
    A["管理员登录综合管理系统"] --> B["查询用户、角色或权限"]
    B --> C{"管理操作类型"}
    C -- "用户管理" --> D["新增或修改用户基础信息"]
    C -- "账号状态" --> E["启用或停用账号"]
    C -- "角色权限" --> F["维护角色和权限关系"]
    C -- "主要业务角色" --> G["为用户设置主要业务角色"]
    D --> H["记录操作日志"]
    E --> H
    F --> H
    G --> H
```

约束：

- MVP 阶段一个账号只有一个主要业务角色。
- 首期不实现复杂多角色切换。
- 账号启停和权限变更等关键操作必须记录日志。

## 16. 建模约束记录

- 医生端继续保持轻量化。
- 不加入医生端 AI 诊断、AI 处方或自动医疗决策。
- AI 仅用于护理培训辅助。
- AI 题目必须基于已入库培训资料生成草稿。
- 只有 AI 题目草稿必须经培训管理员审核后才能进入正式题库或考核。
- MVP 不实现智能派单。
- 不实现真实支付和真实短信。
- 本文档不创建接口、数据库表或工程目录。
