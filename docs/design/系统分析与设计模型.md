## 3. 登录和权限校验时序图

```mermaid
sequenceDiagram
    participant U as 用户
    participant FE as 前端
    participant Auth as Auth模块
    participant DB as 用户/角色/权限数据
    participant Biz as 业务模块
    U->>FE: 输入账号和密码
    FE->>Auth: 提交登录请求
    Auth->>DB: 查询用户、主角色和权限
    DB-->>Auth: 返回账号状态和权限
    Auth-->>FE: 返回Token和当前用户信息
    FE->>Biz: 携带Token访问业务接口
    Biz->>Auth: 校验认证和RBAC功能权限
    Auth-->>Biz: 允许或拒绝
    Biz->>Biz: 在业务Service内校验领域数据权限
    Biz-->>FE: 返回业务结果或错误码
```

## 4. 培训学习和考核时序图

```mermaid
sequenceDiagram
    participant C as 护工/护理人员
    participant FE as 前端
    participant Train as Training模块
    participant DB as 培训数据
    C->>FE: 浏览已发布资源
    FE->>Train: 查询培训类别、标签和资源
    Train->>DB: 读取已发布资源
    DB-->>Train: 返回资源列表
    Train-->>FE: 返回资源
    C->>FE: 学习文章/视频/PPT
    FE->>Train: 提交学习访问记录和时长
    Train->>DB: 更新整体学习时长和最近学习时间
    C->>FE: 提交考核答案
    FE->>Train: 创建考试记录和答案
    Train->>DB: 自动评分单选题和判断题，保存考核记录
    DB-->>Train: 返回本次考核状态
    Train-->>FE: 返回已通过/未通过
```

## 5. 订单预约、分配和执行时序图

```mermaid
sequenceDiagram
    participant F as 老人/家属
    participant O as 运营人员
    participant C as 护工
    participant Care as Care模块
    participant DB as 护理订单数据
    F->>Care: 提交护理服务预约
    Care->>DB: 创建订单，状态=待分配
    O->>Care: 人工分配护工
    Care->>DB: 订单状态 待分配->待确认
    C->>Care: 查看分配给自己的订单
    Care->>Care: 通过CareOrderAccessService校验本人订单范围
    Care->>DB: 查询分配给当前护工的订单
    C->>Care: 确认服务
    Care->>DB: 待确认->已确认
    C->>Care: 开始服务
    Care->>DB: 已确认->服务中
    C->>Care: 完成服务并填写记录
    Care->>DB: 保存服务记录，服务中->已完成
```

## 6. 医生授权老人并访问健康数据时序图

```mermaid
sequenceDiagram
    participant M as 管理员/健康管理人员
    participant D as 医生
    participant Doctor as Doctor模块
    participant Auth as Auth模块
    participant DB as 健康数据
    M->>Doctor: 维护医生老人授权
    Doctor->>DB: 创建或取消授权记录
    Doctor->>DB: 写入操作日志
    D->>Doctor: 查看老人健康档案
    Doctor->>Auth: 校验医生角色和功能权限
    Auth-->>Doctor: 功能权限通过
    Doctor->>Doctor: 通过DoctorAuthorizationService校验授权老人范围
    Doctor->>DB: 查询健康记录、预警、随访和评估
    DB-->>Doctor: 返回授权老人健康数据
```

## 7. AI题目草稿生成、审核和进入正式题库时序图

```mermaid
sequenceDiagram
    participant T as 培训管理员
    participant Train as Training模块
    participant AI as AI适配层
    participant DB as 培训和题库数据
    T->>Train: 选择已入库培训资料
    Train->>AI: 请求生成题目草稿
    AI-->>Train: 返回题目草稿
    Train->>DB: 保存AI草稿和多资料来源关系，状态=DRAFT
    T->>Train: 审核草稿
    alt 审核通过
        Train->>DB: 创建正式题目和选项，草稿=APPROVED
    else 驳回
        Train->>DB: 草稿=REJECTED，记录意见
    end
```
