### 2.3 分层职责

| 层 | 职责 |
|---|---|
| Controller | HTTP 路由、请求校验、调用 Service、统一响应 |
| DTO | 接收外部输入，不暴露 Entity |
| Service | 权限、状态流转、事务和业务规则 |
| Mapper | MyBatis-Plus 数据访问 |
| Entity | 数据库映射 |
| VO | 对外响应结构 |
| Support/Assembler | 访问策略、文本处理和批量组装 |

跨模块协作通过 Service 接口，禁止在 training 中直接引用 file 或 auth 的 Mapper。

### 2.4 统一能力

- `ApiResponse<T>`：统一业务响应。
- `PageResponse<T>`：统一分页结构。
- `GlobalExceptionHandler`：参数、业务和系统异常映射。
- `SecurityConfig`：认证、CORS 和路径权限。
- `HealthController`：`GET /api/v1/health`。
- `FileStorageService`：文件存储抽象。
- `OperationLogService`：关键操作日志。

## 8. 本地启动

### 一键启动

```powershell
$env:DB_PASSWORD = "本机MySQL密码"
./scripts/start-lite.ps1
```

启动：

- 后端 8080。
- 管理端 5173。
- 护工端 5174。
- 门户 5175。

停止：

```powershell
./scripts/stop-lite.ps1
```

### 分别启动

```text
cd backend && mvn spring-boot:run
cd frontend/admin-web && npm run dev
cd frontend/mobile-web && npm run dev
cd frontend/portal-web && npm run dev -- --host 0.0.0.0 --port 5175
```

## 9. 构建与检查

cd backend && mvn verify

cd frontend/portal-web && npm run build

`mvn verify` 包含自动化测试、打包和 Checkstyle。

## 10. 复杂度控制

- 一个 Service 只处理一个清晰子领域。
- Controller 不写业务事务。
- 批量查询题目、选项、标签和成绩，避免 N+1。
- 跨模块使用 Service。
- 大型组装逻辑拆到 Assembler/Support。
- 新增通用常量时按领域拆分，不建立混合状态“大杂烩”类。
- 前端请求、会话和反馈使用公共封装，不在页面重复实现。

## 11. 当前限制

- 没有正式 Docker/nginx 部署工程。
- 文件存储为本地实现。
- Token 黑名单为单机内存实现。
- Redis、消息队列和对象存储不是强制依赖。
- PowerDesigner 模型文件尚未生成。
- 历史文档中的完整版工程结构不适用于轻量版。
