# 原型设计

项目名称：CareNexus 颐联  
版本：轻量版 2.0  
更新时间：2026-07-15

## 1. 原型范围

当前前端由三个工程组成：

- `portal-web`：统一门户与品牌入口。
- `admin-web`：管理员 PC 工作台。
- `mobile-web`：护工移动学习端。

本说明基于当前路由和页面代码，不再描述护理订单、医生健康管理或已删除的待接入页面。

## 2. 视觉与交互原则

- 面向养老护理培训，整体清晰、稳重、易读。
- 管理端优先信息密度和操作效率。
- 护工端优先移动设备触控、课程卡片和学习进度反馈。
- 统一门户承担品牌展示和角色分流。
- 页面必须提供加载、空数据、错误、无权限和操作反馈状态。
- 不在正式界面展示权限码、接口待接入说明或开发注释式文案。

## 3. 统一门户

### 3.1 首页

主要区域：

1. CareNexus 品牌与产品定位。
2. 管理员和护工入口。
3. 培训能力介绍。
4. 课程展示与平台价值。
5. 登录后的角色分流。

```text
┌──────────────────────────────────────────┐
│ CareNexus 颐联                     登录 │
├──────────────────────────────────────────┤
│ 面向养老护理人员的培训与AI学习平台       │
│ [管理员工作台]       [护工学习端]         │
├──────────────────────────────────────────┤
│ 课程管理  逐课考核  学习笔记  AI辅助      │
├──────────────────────────────────────────┤
│ 精选课程 / 平台介绍 / 使用说明             │
└──────────────────────────────────────────┘
```

### 3.2 课程工作区

门户中可承载课程内容、富文本笔记和学习辅助组件。页面应与后端当前用户和课程数据保持一致，不使用假数据替代接口。

## 4. 管理员端信息架构

```text
管理工作台
├── 首页概览
├── 培训资源
│   ├── 资源列表
│   ├── 新增资源
│   ├── 资源详情
│   └── 编辑资源
├── 分类与标签
├── 题库与考核
├── AI 草稿审核
└── 培训成绩
```

### 4.1 管理工作台

展示当前系统的培训管理入口和关键概览，不展示已经删除的订单、医生或健康模块。

### 4.2 培训资源列表

建议布局：

```text
┌────────────────────────────────────────────┐
│ 培训资源                       [新增资源]   │
├────────────────────────────────────────────┤
│ 搜索  类型  分类  标签  状态                │
├────────────────────────────────────────────┤
│ 封面 │ 标题 │ 类型 │ 分类 │ 状态 │ 操作    │
│      │      │      │      │      │详情/编辑│
├────────────────────────────────────────────┤
│                    分页                     │
└────────────────────────────────────────────┘
```

操作根据状态显示：

- 草稿：编辑、发布。
- 已发布：详情、下架。
- 已下架：编辑、重新发布。

### 4.3 资源编辑页

字段：资源类型、存储方式、标题、摘要、封面、类别、标签、课程时长、文本内容/本地文件/外部链接。

交互要求：

- 存储方式变化时只显示相关输入项。
- 上传封面后立即预览。
- 表单校验失败时定位到具体字段。
- 已发布资源不直接进入可编辑状态。

### 4.4 分类与标签

采用列表或左右分栏：

- 名称、排序、状态。
- 新增和编辑弹窗。
- 启用/停用二次确认。

### 4.5 题库与考核

- 按课程查看考核。
- 显示考核状态、通过分、最大尝试次数和题目数量。
- 支持单选题、判断题编辑。
- 单选题编辑器必须确保一个正确选项。
- 发布前检查题目和分值配置。

### 4.6 AI 草稿审核

```text
┌──────────────────────────────────────────┐
│ AI草稿审核      状态筛选  [生成草稿]     │
├──────────────────────────────────────────┤
│ 草稿类型 │ 来源课程 │ 状态 │ 创建时间    │
├──────────────────────────────────────────┤
│ 详情区：题干、选项、答案、解析、来源      │
│ [审核通过] [驳回]                        │
└──────────────────────────────────────────┘
```

应明确提示：审核通过只进入题库草稿，不自动发布或加入考核。

### 4.7 培训成绩

按护工显示：姓名、课程总数、通过课程数、平均分、最高分和整体培训状态。支持展开逐课成绩。

## 5. 护工端信息架构

```text
护工学习端
├── 登录
├── 移动工作台
├── 培训资源
│   └── 课程详情
│       ├── 学习内容
│       ├── 课程考核
│       ├── 错题
│       ├── 笔记
│       ├── 讨论
│       └── 作业
├── 学习进度/成绩
├── 学习笔记
└── 我的账号
```

### 5.1 登录页

- 用户名、密码和登录按钮。
- 登录失败统一提示，不暴露账号是否存在。
- 已有有效会话时自动进入工作台。

### 5.2 移动工作台

展示学习入口、当前课程情况、成绩摘要和个人账号入口。使用卡片和大触控区域，避免桌面端密集表格。

### 5.3 课程列表

- 只展示已发布课程。
- 课程卡包含封面、标题、摘要、分类、标签、建议时长和通过状态。
- 支持搜索和筛选。

### 5.4 课程详情

建议采用页内标签或分区：

```text
[课程内容] [考核] [错题] [笔记] [讨论] [作业]
```

课程内容根据资源存储方式展示文本、文件或外部链接。记录有效学习访问，避免仅打开页面即虚增大量时长。

### 5.5 课程考核

- 题目逐题或列表展示。
- 单选题使用单选控件。
- 判断题使用“正确/错误”。
- 提交前提示未答题目。
- 提交后显示得分、通过状态、答案和解析。

### 5.6 学习进度

页面实际定位为课程成绩仪表盘：

- 每课最高分、平均分、通过状态。
- 已通过课程数/课程总数。
- 整体培训状态。
- 点击课程进入详情或错题。

### 5.7 富文本笔记

- 固定工具栏。
- 居中的白色纸张式编辑区。
- 独立标题和正文。
- 支持图片、表格、待办、链接等常用格式。
- 自动处理空白笔记与保存反馈。

### 5.8 课程讨论

- 主题列表、回复列表和嵌套回复。
- 支持最新等排序方式。
- 点赞状态即时反馈。
- 只有本人内容显示删除入口。

### 5.9 课后作业

- 展示作业题目、类型和截止时间。
- 提交答案后显示提交状态。
- 避免重复点击造成多条提交。

## 6. 权限与异常页面

三个前端均应统一处理：

- 401：清除会话并跳转登录或门户。
- 403：显示无权访问页。
- 404：显示页面不存在。
- 后端不可用：保留用户可理解的错误提示，不伪造成功数据。
- 空数据：说明当前没有课程、成绩、草稿、讨论或作业。

## 7. 响应式要求

- 管理端主要面向 1280px 及以上桌面浏览器。
- 护工端优先 360–430px 移动宽度，也应在桌面浏览器可用。
- 门户适配桌面和移动端。
- 表格在窄屏下转为卡片、横向滚动或减少次要列。

## 8. 已删除原型

以下旧页面不属于轻量版：

- 护理服务浏览、预约下单、订单分配与执行。
- 老人、家属和地址管理。
- 评价、投诉和服务记录。
- 医生授权、健康记录、预警、随访和健康评估。
- “待接入模块”占位页面。

## 9. 验收标准

- 页面路由与当前代码一致。
- 菜单不出现已删除角色和模块。
- 所有关键页面使用真实后端接口。
- 构建和 lint 通过。
- 登录、权限、空状态、异常和重复提交有明确反馈。
- 管理端、护工端和门户之间的入口与返回路径清晰。
