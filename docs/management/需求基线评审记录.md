# Lite需求范围确认记录

原 `inception-v1.0` 四端需求基线属于历史阶段。项目负责人后续明确选择轻量方案，当前有效范围以 `docs/management/LITE_SCOPE.md` 和 `docs/requirements/软件需求规约.md` 为准。

## 当前确认

- 产品：护理培训平台。
- 角色：管理员、护工。
- 主流程：资料管理、在线学习、互动笔记、作业考试、成绩错题、AI辅助。
- AI边界：仅护理培训，不提供医疗决策；草稿必须审核。
- 集成分支：`lite_develop`。

旧老人/家属、医生、健康管理和护理订单范围不进入 Lite 验收。若未来恢复，必须重新进行需求澄清、架构评审和数据库变更确认。
