# 轻量版需求基线评审记录

项目名称：CareNexus 颐联  
评审版本：Lite 2.0  
评审日期：2026-07-15  
目标分支：`lite_develop`

## 1. 评审背景

项目早期 CareNexus 需求基线 v1.0 覆盖培训、护理订单、老人家属和医生健康管理。2026-07-11 项目建立 `lite_develop`，范围收敛为管理员与护工培训平台。原基线作为历史过程保留，但已不能直接作为轻量版验收依据。

本次评审针对重写后的现行文档，核对其是否与 `lite_develop` 当前代码、路由和数据库脚本一致。

## 3. 评审结论

| 检查项 | 结论 |
|---|---|
| 角色范围 | 通过：仅 ADMIN、CAREGIVER |

| 前端范围 | 通过：门户、管理端、护工端 |
| 数据库口径 | 通过：001–008 最终 28 张表 |

| 测试范围 | 通过：已改为轻量版核心用例 |
| 历史材料处理 | 通过：Changelog、Test Log 和历史任务不回写 |

## 4. 关键变更

相对原 v1.0 基线：

- 删除老人、家属、医生、健康管理人员、运营人员和独立培训管理员。
- 删除护理预约订单、评价投诉和医生健康管理。
- 管理员承担培训管理职责。
- 保留并扩展逐课考核、成绩和错题。
- 新增现行基线中的富文本笔记、讨论、点赞和课后作业。
- AI 从“适配骨架/稳定 Mock”更新为 Mock 与 DeepSeek 可切换实现。
- 数据库从完整版 36 张表收敛并扩展为轻量版 28 张表。
- 三个前端从骨架/待接入说明更新为当前页面结构。

## 5. 仍需验证事项

文档一致性评审通过不等于软件全量验收通过。以下事项仍保持待执行：

1. 干净 MySQL 8 环境顺序执行 001–008。
2. 最终表数、外键、唯一约束和演示数据核验。
3. 后端 `mvn verify`。
4. 两个 Vue 前端 lint/build 和 React 门户 build。
5. 管理员、护工核心 HTTP 主链路。
6. 讨论、作业、笔记和 DeepSeek 模式回归。
7. multipart 全局大小与文档/视频业务限制一致性核对。
8. CDM/PDM、部署说明、用户手册和最终测试报告。

## 6. 评审决定

形成 **CareNexus 轻量版需求基线 Lite 2.0**。

- 允许后续测试、缺陷修复和交付以本基线开展。
- 轻量版功能变化必须同步修改现行文档和测试。
- 恢复完整版模块或新增医疗能力属于重大范围变更，必须重新评审并新增 ADR。
- 原 CareNexus v1.0 基线继续保留在 Git 历史中，不删除、不改写。

本次文档通过 PR 合并到 `lite_develop` 后，以合并提交 SHA 作为 Lite 2.0 文档基线标识。完成软件全链路回归后，再创建发布候选标签。
