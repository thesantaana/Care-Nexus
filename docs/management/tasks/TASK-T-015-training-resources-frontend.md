## 实现范围

- PC 端：培训资源列表、筛选、分页、详情、新增、编辑、发布、下架，以及类别和标签维护。
- 移动端：已发布资源浏览、资源详情、学习访问上报和个人学习进度。
- 仅调用现有 `/api/v1/training/**` 接口；管理操作按 `training:resource:manage` 控制，学习端按 `training:resource:view` 控制。

## 不实现范围

- 不使用业务 Mock 数据或硬编码资源、考核 ID。
- 不实现本地文件上传、下载或预览；现有后端没有文件 HTTP 接口。
- 不实现考核列表、题库列表、成绩历史或考试页面；现有后端缺少完成这些页面所需的查询接口。
- 不实现订单、医生健康管理、完整综合管理或 AI 培训功能。

## 当前验证记录

- 两个前端 `npm run lint` 和 `npm run build` 已通过。
- 本机 Vite 开发服务器可启动并返回两个应用入口 HTML。
- 真实 training HTTP 联调待本机后端环境可用后执行；当前 MySQL 服务未运行，且 Maven 依赖下载受 TLS 握手失败阻塞。

## 验收标准

- 培训管理员可以基于真实接口维护类别、标签和 TEXT/EXTERNAL_LINK 培训资源，并完成发布和下架。
- 护工/护理人员仅能看到已发布资源，无法进入管理操作。
- 学习访问与学习进度使用真实接口，且提交期间防止重复操作。
- 角色、权限、加载、空、错误和无权限状态清晰可见。
- 两个前端 lint/build 通过；真实 HTTP 联调完成后再更新为 `REVIEW`。
