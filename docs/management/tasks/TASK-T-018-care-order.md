## 目标用户

- 老人、家属：绑定老人、浏览服务、维护地址、预约、查看和取消订单、评价及投诉。
- 运营人员：维护服务项目、查询待分配订单、人工分配护工、处理投诉。
- 护工/护理人员：查看本人订单，确认、开始和完成服务并填写服务记录。

## 实现范围

- 家属使用老人账号标识和独立绑定码建立绑定，绑定码只保存 BCrypt 哈希。
- 服务项目管理和老人/家属服务浏览。
- 地址新增、修改、停用、默认地址维护。
- 护理预约、订单查询、取消、人工分配和护工执行。
- 已完成订单的一次评价和一次投诉，运营人员完成投诉处理。
- 所有关键状态变化记录订单状态日志和操作日志。

## 权限和数据范围

- 家属只能访问已绑定老人；老人只能访问本人档案和相关订单。
- 护工只能查看和操作分配给自己的订单。
- 运营人员管理服务项目、待分配订单、人工派单和投诉。
- 管理员通过既有角色全权限映射获得护理模块权限。

## 状态流转

```text
待分配 -> 待确认 -> 已确认 -> 服务中 -> 已完成
待分配 / 待确认 / 已确认 -> 已取消
投诉：处理中 -> 已处理
```

评价和投诉独立于订单主状态。

## 业务规则

- 预约时间必须晚于当前时间，不实现真实支付。
- 派单对象必须是正常状态的护工账号；MVP 不实现智能派单、排班冲突和时间重叠检查。
- 评分范围为 1 至 5；每个订单最多评价一次、投诉一次。
- 完成服务必须填写服务内容，完成时间不得晚于提交操作的合理当前时间。
- 地址默认状态由 Service 层保证同一地址范围只有一个有效默认地址。
