# Task Documents

本目录用于存放经过需求澄清和用户确认后的单项任务文档，也允许在需求澄清阶段创建草稿任务文档。

## 命名规则

```text
TASK-xxx.md
```

示例：

```text
TASK-001-确认项目分工与需求基线.md
```

## 任务状态

- `DRAFT`：需求澄清阶段的草稿任务文档，尚未获得用户最终确认。
- `CONFIRMED`：用户已明确确认需求和验收标准，可以开始实施。
- `IMPLEMENTING`：正在实施中。
- `REVIEW`：已形成结果，等待组长、用户或相关负责人审核。
- `DONE`：已完成、已验证，并且过程记录已更新。
- `CHANGED`：需求已发生变更，需要保留原始决策痕迹并记录变更内容。

需求澄清阶段可以创建 `DRAFT` 任务文档。只有用户明确授权后，任务才能改为 `CONFIRMED`，并进入编码或具体实施阶段。

## 内容要求

任务文档至少记录：

- 任务名称
- 需求来源
- 目标用户
- 业务范围
- 主流程
- 异常流程
- 权限规则
- 状态流转
- 数据库变化
- 接口变化
- 页面范围
- 测试范围
- 验收标准
- 不实现内容
- 用户确认时间
- 后续需求变更

## 使用规则

涉及业务、架构、页面、接口、数据库、权限和状态流转的开发任务，必须先完成需求澄清，并在用户明确批准实施后再从 `DRAFT` 转为 `CONFIRMED`。
