# CareNexus 轻量版范围

更新时间：2026-07-15  
开发基线：`lite_develop`

## 1. 产品定位

CareNexus 轻量版是面向养老护理人员的培训、考核和 AI 辅助学习平台。系统通过统一门户连接管理员工作台与护工学习端，不承载护理订单或医生健康管理业务。

## 2. 保留角色

| 角色 | 代码 | 职责 |
|---|---|---|
| 管理员 | `ADMIN` | 账号与权限基础、培训资源、分类标签、题库考核、AI 草稿审核、培训成绩查看 |
| 护工/护理人员 | `CAREGIVER` | 课程学习、考核答题、成绩与错题、笔记、讨论、作业和个人账号 |

管理员吸收原培训管理员与运营人员中和培训有关的职责。系统不再保留独立培训管理员角色。

## 3. 保留能力

### 后端

- JWT 登录、退出、当前用户与 RBAC。
- 培训类别、标签、文章、视频、PPT 资源。
- 文本、本地文件、外部链接三种资源存储方式。
- 课程学习访问与学习记录。
- 单选题、判断题、逐课考核、自动评分、成绩汇总和错题。
- 富文本课程笔记与图片上传。
- 课程讨论、回复、嵌套回复、点赞和本人内容删除。
- 课后作业与个人提交。
- AI 问答、总结、学习建议、题目草稿和审核。
- 文件存储与操作日志。

### 前端

- React 统一门户。
- Vue 管理员 PC 端。
- Vue 护工移动端。

### 数据库

当前最终结构共 28 张表：

- 权限与基础数据：5 张。
- 审计与文件：2 张。
- 培训资源、学习、考核和 AI：15 张。
- 课程互动与作业：4 张。
- 讨论点赞：2 张。

数据库以 `database/init/001` 至 `008` 的顺序脚本为准。

## 4. 删除范围

以下内容已从轻量版代码、数据库、测试、前端路由和现行文档中移除：

- 医生、健康管理人员、老人、家属、运营人员和独立培训管理员角色。
- 家属绑定、地址、服务项目、护理预约、订单分配、护工执行、取消、评价和投诉。
- 医生老人授权、健康档案、健康记录、预警、随访、干预和健康评估。
- 医疗诊断、处方、个体化治疗方案和自动医疗决策。

## 5. AI 边界

- 仅使用调用者有权访问的文本培训资料。
- 普通护工只能使用已发布资源。
- 输出保留来源引用，资料不足时明确说明。
- 题目草稿仅支持单选题和判断题。
- 审核通过只创建 `DRAFT` 正式题目，不自动发布或加入考核。
- 默认使用 Mock，可通过配置切换 DeepSeek。
- 不读取健康数据，不提供医疗判断。

## 6. 分支关系

- `develop` 保留完整版历史实现。
- `lite_develop` 是轻量版唯一集成基线。
- 轻量版后续功能、修复和文档更新均以 `lite_develop` 为目标分支。
- 不从 `develop` 整体合并回轻量版；需要复用时只挑选明确且不违反轻量范围的提交。

## 7. 文档口径

现行范围、需求、设计、接口和数据库文档以本文件及 `lite_develop` 当前代码为准。`CHANGELOG.md`、`TEST_LOG.md`、个人日志和历史任务文件保留完整版开发过程，仅作为审计记录，不代表轻量版当前功能范围。
