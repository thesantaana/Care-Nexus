# ADR-001 Lite总体架构与工程结构

状态：已采用

## 决策

CareNexus Lite 使用一个 Spring Boot 模块化单体后端、一个 MySQL 数据库和三个独立前端（门户、管理员端、护工端）。当前产品只保留护理培训主线和管理员/护工角色。

## 原因

- 课程周期和团队规模不适合微服务。
- 培训内容、学习、考试和AI草稿需要一致事务与快速联调。
- 管理和学习界面工作方式不同，独立前端比单一巨型工程更清晰。
- 门户包含较强品牌和富文本交互，保留 React；业务工作台沿用 Vue。

## 结果

- 删除旧医生、老人、家属和护理订单交付范围。
- `lite_develop` 为当前集成分支。
- AI、文件和认证保留适配边界，避免业务直接绑定外部实现。
