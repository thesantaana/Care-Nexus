# 测试计划

项目名称：CareNexus 颐联  
版本：轻量版 2.0  
更新时间：2026-07-15  
测试基线：`lite_develop`

## 1. 测试目标

验证管理员与护工培训主线在干净环境中可构建、可启动、可联调、权限正确、数据落库一致，并确认 AI、笔记、讨论、作业和逐课考核不越过产品安全边界。

本计划不再测试护理订单、老人家属、医生健康管理等已从轻量版删除的业务。

## 2. 测试范围

### 包含

- 登录、退出、当前用户、JWT、账号状态和 RBAC。
- 培训分类、标签、资源、封面、发布与下架。
- 课程浏览、学习访问和个人学习记录。
- 单选题、判断题、逐课考核、自动评分、成绩和错题。
- 富文本笔记及图片上传。
- 课程讨论、回复、嵌套回复、点赞和删除。
- 课后作业与个人提交。
- AI 问答、总结、建议、题目草稿和审核。
- 管理员端、护工端和统一门户。
- 001–008 数据库脚本和 28 张表结构。
- 文件上传、安全错误和操作日志。

### 不包含

- 护理预约订单和医生健康管理。
- 原生 App、小程序。
- 正式 Docker/Kubernetes 部署。
- 医疗诊断正确性。
- 大规模生产性能和高可用测试。

## 3. 测试类型

| 类型 | 目标 |
|---|---|
| 单元测试 | Service 状态规则、评分、AI 草稿和数据组装 |
| 接口测试 | Controller 权限、参数、状态码和响应结构 |
| 数据库测试 | 001–008 顺序导入、表数、约束、种子和演示数据 |
| 集成测试 | 后端连接真实 MySQL 的核心 HTTP 主链路 |
| 前端静态检查 | ESLint、TypeScript 和构建 |
| 前端冒烟 | 三个入口、登录、路由守卫和关键页面 |
| 安全测试 | 401/403、账号状态、文件上传、敏感信息和越权 |
| 回归测试 | 修复后验证认证、培训、考核和 AI 主线不受影响 |
| 兼容性测试 | Chrome/Edge，常见桌面与移动宽度 |

## 8. 核心 HTTP 主链路

### 管理员主线

1. 登录。
2. 查询当前用户。
3. 新增分类和标签。
4. 创建草稿课程。
5. 发布课程。
6. 创建题目和考核。
7. 发布考核。
8. 生成并审核 AI 草稿。
9. 查看护工成绩。
10. 下架课程。
11. 退出并验证旧 Token 401。

### 护工主线

1. 登录。
2. 查询已发布课程。
3. 查看课程详情。
4. 记录学习访问。
5. 获取并提交逐课考核。
6. 查看成绩和错题。
7. 保存和读取富文本笔记。
8. 发布讨论、回复、点赞和删除本人内容。
9. 查看作业并提交。
10. 使用 AI 问答、总结和建议。

## 9. 权限矩阵

| 操作 | 未登录 | 护工 | 管理员 |
|---|---:|---:|---:|
| 查询已发布课程 | 401 | 200 | 200 |
| 创建/发布资源 | 401 | 403 | 200 |
| 参加考核 | 401 | 200 | 按业务限制 |
| 查看本人成绩 | 401 | 200 | 个人范围 |
| 查看护工成绩列表 | 401 | 403 | 200 |
| 管理个人笔记 | 401 | 200 | 个人范围 |
| 讨论和作业 | 401 | 200 | 按权限参与 |
| AI 问答 | 401 | 200 | 200 |
| AI 草稿生成/审核 | 401 | 403 | 200 |

还需测试护工不能访问草稿或下架资源，以及不能读取他人笔记、成绩、错题和作业提交。

## 10. AI 测试

### Mock

- 输出稳定且包含来源引用。
- 下架、非文本或无权访问资料被拒绝。
- 草稿只支持单选和判断。
- 审核通过创建 `DRAFT` 正式题目。
- 驳回不创建题目。
- 重复审核返回 409。

### DeepSeek

- Key 不存在时启动或请求错误清晰。
- 只根据资料回答。
- 资料不足时说明不足。
- 不输出诊断、处方和治疗方案。
- 外部错误不泄露 Key 或完整请求上下文。

## 11. 文件测试

- 合法 jpg/png/webp 封面和笔记图片。
- 合法 pdf/ppt/pptx/doc/docx/mp4/webm 资料。
- 超限文件。
- 扩展名与 MIME 不一致。
- 被禁止的 exe、bat、cmd、sh、js、jsp、php。
- 文件名路径穿越和特殊字符。
- 响应不包含服务器绝对路径。

## 12. 前端验收

- 三个入口可访问。
- 角色分流正确。
- Token 自动携带。
- 401 清理会话，403 显示无权页。
- 管理菜单不出现订单或医生模块。
- 护工页面不出现管理操作。
- 所有关键页面使用真实 API。
- 加载、空数据、失败和重复提交反馈完整。
- 移动端 360–430px 可操作。

## 13. 缺陷管理

严重级别：

- P0：数据泄露、权限绕过、核心系统不可用。
- P1：登录、课程、考核或成绩主线无法完成。
- P2：笔记、讨论、作业、AI 或主要页面异常。
- P3：文案、样式和低影响体验问题。

交付条件：

- P0 为 0。
- P1 为 0。
- P2 有明确处理结论和回归记录。
- 构建、数据库导入和核心 HTTP 主线通过。

## 14. 记录规则

实际执行结果追加到 `TEST_LOG.md`。历史完整版测试记录保留，不改写。每轮记录必须包含：

- 日期和提交 SHA。
- 环境版本。
- 执行命令。
- 通过/失败数量。
- 错误摘要和处理。
- 未执行事项及原因。

没有执行的测试不得记录为通过。
