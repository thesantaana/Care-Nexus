# 项目开发计划

项目名称：CareNexus 颐联  
版本：轻量版 2.0  
更新时间：2026-07-15

## 1. 项目目标

在现有 `lite_develop` 基础上完成一套可演示、可测试、可交付的养老护理培训平台，形成“管理员发布课程与考核—护工学习与答题—成绩与错题反馈—笔记、讨论和作业巩固—AI 辅助学习与出题审核”的闭环。

## 2. 产品范围

### 包含

- 管理员、护工两角色。
- 统一门户、管理员端和护工端。
- 认证、RBAC、培训资源、逐课考核、成绩和错题。
- 富文本笔记、讨论、点赞和课后作业。
- Mock/DeepSeek AI 培训辅助。
- MySQL 8 数据库与本地文件存储。

### 排除

- 护理订单、老人家属和医生健康管理。
- 医疗诊断、处方和治疗决策。
- 原生 App、小程序和正式微服务化。
- 当前周期内的高可用和大规模生产部署。

## 3. 当前基础

截至 2026-07-15：

- 后端核心能力已实现。
- 三个前端工程已形成主要页面。
- 8 个数据库脚本已覆盖 28 张表及演示数据。
- 启动/停止脚本已存在。
- 当前主要工作转为文档归一、全链路回归、交付材料和版本发布。

## 5. 工作分解

### 阶段 A：范围与文档归一

- 更新 README 和 GitHub 项目说明。
- 统一轻量版范围、状态和任务路线。
- 重写需求、用例、流程、架构、数据库、API 和测试文档。
- 保留历史 Changelog/Test Log，不篡改过程。

交付物：现行文档基线 2.0。

### 阶段 B：环境与数据库验证

- 在干净 MySQL 8 环境执行 001–008。
- 验证 28 张表、外键、唯一约束、角色、账号和演示数据。
- 检查 UTF-8 中文数据。
- 核对 multipart 大小配置与实际文件能力。

交付物：数据库验证记录和缺陷清单。

### 阶段 C：构建与自动化回归

- 后端 `mvn verify`。

- 门户 TypeScript/build。
- 修复编译、静态检查和回归问题。

交付物：构建记录和自动化测试结果。

### 阶段 D：真实 HTTP 联调

- 护工学习、考试、错题、笔记、讨论、作业和 AI 主线。
- 文件上传和权限矩阵。
- 三个前端和后端真实接口联调。

交付物：HTTP 联调记录、缺陷和回归结果。

### 阶段 E：交付完善

- 部署说明。
- 管理员用户手册。
- 护工用户手册。
- 最终测试报告。

- 演示数据说明、答辩 PPT 和项目总结。

交付物：完整交付包。

### 阶段 F：发布

- 冻结轻量版范围。

## 7. 质量目标

- P0、P1 缺陷为 0。
- 后端 verify 通过，Checkstyle 0 violations。
- 两个 Vue 前端 lint/build 通过。
- React 门户 build 通过。
- 干净 MySQL 8 导入成功，最终表数 28。
- 未登录 401、越权 403、状态冲突 409 语义正确。
- 演示主线无假数据替代真实接口。
- AI 不越过培训资料和医疗安全边界。

## 8. 风险与应对

| 风险 | 影响 | 应对 |
|---|---|---|
| 历史文档和当前代码冲突 | 开发、答辩误解 | 建立 2.0 现行文档并标注历史记录 |
| MySQL 本机服务不稳定 | 联调受阻 | 使用同版本隔离实例或干净测试库 |
| Maven/前端依赖网络失败 | 构建中断 | 预先缓存依赖并记录环境问题 |
| 三个前端会话不一致 | 登录跳转异常 | 统一 Token、角色和 401 处理约定 |
| multipart 全局限制过小 | 大文件无法上传 | 核对配置、需求和测试后统一调整 |
| DeepSeek 网络或 Key 不可用 | AI 演示失败 | 默认 Mock，真实模型作为可选增强 |
| 演示数据脚本非幂等 | 重复导入失败 | 使用全新库验证并明确脚本使用方式 |
| 重新引入完整版需求 | 范围失控 | 任何范围变化先更新 LITE_SCOPE 和 ADR |

## 9. 验收主线

管理员：登录→创建课程→发布→创建考核→发布→生成/审核 AI 草稿→查看成绩。

护工：登录→学习三门课程→逐课考试→查看成绩/错题→记笔记→讨论→提交作业→使用 AI 辅助。

## 11. 变更原则

开发计划只针对轻量版。护理订单或医生健康管理如需恢复，应作为新版本重新评审，不作为本计划中的“顺手补回”功能。
