# CareNexus Lite 配置管理计划

## 分支

- `main`：历史稳定基线。
- `lite_develop`：Lite日常集成和演示分支。
- `feature/*`、`fix/*`、`docs/*`：从 `lite_develop` 创建，通过审核后合回。

已删除旧 `develop`。不得在文档或脚本中继续把它作为当前集成分支。

## 配置项

源代码、SQL、API/需求/设计/测试文档、启动脚本和前端静态资源均受Git管理。`target`、`node_modules`、`dist`、上传文件、日志、数据库密码、Token和AI密钥不得提交。

## 版本与变更

提交信息使用 Conventional Commit 前缀。需求、API或数据库变化必须同步对应文档/SQL并执行适用验证。Lite发布标签在综合回归完成后创建，当前不提前标记发布版本。

## 回滚

优先使用新提交修复或 `git revert`；禁止覆盖他人工作或使用破坏性重置。数据库结构变更需提供兼容或回滚说明，演示数据脚本需可重复执行。
